DB

1. 데이터베이스(Database, DB)란 무엇인가?
데이터베이스란 단순히 정보를 저장해두는 공간을 의미하는 것이 아니라, 다양한 정보들이 체계적으로 구성되고, 여러 사용자나 시스템이 동시에 접근하거나 공유할 수 있도록 설계된 정보의 집합체이며; 이 데이터베이스는 특정 목적을 가진 애플리케이션이나 서비스를 위해 효율적으로 데이터를 관리할 수 있도록 구조화되어 있다는 점에서 단순한 파일 저장소와는 분명한 차이를 가진다.

현대 사회에서는 우리가 사용하는 대부분의 정보, 즉 스마트폰으로 전송한 메시지나, 소셜 미디어에 올린 사진, 교통카드를 찍을 때 발생하는 로그, 커피 전문점에서 결제한 기록 등 일상 속 거의 모든 디지털 흔적이 데이터베이스에 저장되어 관리되고 있기 때문에, 비단 IT 전공자가 아니더라도 데이터베이스의 개념을 이해하는 것이 매우 중요하다고 볼 수 있다.

2. DBMS(Database Management System)란?
DBMS는 말 그대로 데이터베이스를 생성하고, 구조화하며, 그 위에 CRUD(생성, 읽기, 수정, 삭제)와 같은 작업을 수행할 수 있도록 돕는 운영 및 관리용 소프트웨어를 의미하며; 이는 단순히 데이터를 담는 그릇이 아닌, 데이터베이스라는 복잡한 구조물을 효율적으로 운용하기 위한 핵심 시스템이다.

예를 들어 은행의 예금 시스템을 생각해보면, 수많은 고객의 계좌 정보가 하나의 DB에 저장되어 있고; 이 데이터는 고객 본인, 은행 직원, ATM 기기, 온라인 뱅킹 서비스 등 다양한 채널에서 동시에 접근되며 이러한 동시성 관리, 보안, 백업 등 복잡한 기능이 가능한 이유는 바로 이 DBMS가 데이터베이스를 중재하고 통제하고 있기 때문이다.

1. 데이터베이스 기본 개념
용어	정의
데이터베이스(Database, DB)	여러 사람이 공유하여 사용할 목적으로 통합해 저장한 데이터의 집합
관계형 데이터베이스(RDB)	여러 개의 테이블이 관계를 맺고 있는 구조의 데이터베이스
DBMS (Database Management System)	데이터베이스를 생성, 관리, 운영하는 소프트웨어 시스템
SQL (Structured Query Language)	DBMS에서 데이터를 정의·조작·제어하기 위한 표준 질의 언어


2. 엔티티 및 스키마
용어	정의
엔티티(Entity)	사람, 장소, 사물, 사건 등 현실 세계에서 고유하게 식별 가능한 개체
예: 과목코드 F035인 자료구조
엔티티 집합(Entity Set)	동일한 속성을 가진 엔티티들의 모임
스키마(Schema)	데이터베이스 전체 구조를 정의한 것
데이터 개체, 속성, 관계 등을 명세

6. 데이터베이스 언어(SQL) 종류
분류	전체 명칭	설명	주요 명령어
DDL	Data Definition Language	데이터베이스 구조 정의	CREATE, ALTER, DROP
DML	Data Manipulation Language	데이터 조작 (삽입, 수정, 삭제, 조회)	INSERT, UPDATE, DELETE, SELECT
DCL	Data Control Language	권한 부여, 취소	GRANT, REVOKE
TCL	Transaction Control Language	트랜잭션 제어	COMMIT, ROLLBACK, SAVEPOINT
